HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element is the entire element from opening tag to closing tag
  • A tag is only what appears inside the angle brackets
  • Linking a CSS file via a Link element is done within the Head element
  • Linking a JS file via a Script element is done within the Body element

CSS

  • Inline CSS happens inside the HTML tags
  • Internal CSS style sheet happens within the HTML "style" element
  • External CSS style sheet happens in a seperate file and is linked to the HTML file by using a "link" tag
  • A margin property indicates how much space we want around the outside of an element
  • The padding property adds space around the content inside an element
  • '*' can be used as a wild card to define all elements
  • Giving display the value "block" will make each affiliated element start on a new line
  • Including a period '.' before an HTML element in CSS will designate it as a class
  • For directional elements, the first entry is for top, bottom and the second entry is for left, right (ex. margin: 40px, auto)

Git

General commands:

  • git status: checks what branch we are currently on
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git checkout main: switches to main branch
  • git add [file]: adds file to commit
  • git commit -m "[description]": commits staged content
  • git pull: fetches and merges any commits from repo branch
  • git push origin [branch-name]: transmits local commits to remote repo
  • git config --global user.name "[firstname lastname]": sets a name for commits
  • git config --global user.email "[valid-email]": sets an email associated with name
  • *git config --global color.ui auto: sets automatic cmd line coloring
  • *git log: shows commit history
  • *git stash: saves changes
  • *git rm [file]: deletes file from project
  • *git merge [branch-name]: merge a remote branch into your current one
  • *git reset [file]: unstages a file
  • *git diff: shows what is changed
  • *git branch: lists your branches

Git Flow steps:

  • cd .. (navigate to main directory)
  • git add -A
  • git commit -m "[your message here]"
  • git pull origin main
  • git push origin feature/add-js
  • > code
  • > [branch icon] > feature/[branch-name]

  • > [create pull request]
  • > [merge pull request]

JavaScript

  • Variables in JS are dynamic meaning they aren't directly associated with any particular value type
  • Types include Boolean, Null, Undefined, Number, BigInt, String, and Symbol
  • The keyword 'var' is used to define a variable
  • The tag 'script' has the same role as the tag 'link'
  • Always end code lines with a semicolon: ';'
  • Comments can be created by using // or by surrounding text within '/* */'
  • Equality checks are done using '===' and Inequality checks are likewise done using '!=='
  • Anonymous functions can be written by 'function ()' or '() =>'
  • Event listeners are done using 'addEventListener()'
  • The function 'document.querySelector([tag])' is used to select a tag
  • The function 'alert()' displays a dialog box
  • The function 'prompt()' asks the user for enter data and stores it in a value
  • The function 'console.log()' outputs whatever is contained within the parentheses to the web console. It is mainly used for debugging
  • The property '.length' gives us the array's length
